## 本次实现内容

- 新增 `ComponentId.MATRIX_EFFECT` 与使用 `DotMatrixEffect` 的最短可运行基础示例；示例包含稳定宽高比，避免绝对定位 Canvas 因根容器零高度而无法绘制。
- 在 `componentRegistry` 中注册 Matrix Effect 的名称、说明、分类、版本、标签、无额外依赖和固定三文件站点元信息。
- 使用 `propsInfo` 分别提供 `MatrixEffect`、`DotMatrixEffect`、`AsciiEffect` 三组 API 表；公共播放、帧率、DPR、离屏暂停、Reduced Motion、可访问性、生命周期回调和根容器属性复用同一份元数据。
- API 表逐项对齐真实类型、必填性和有效默认值，明确核心 `clearColor` 与两个预设 `backgroundColor` 的分层，并保留 ASCII 默认字符集开头的空格。
- 新增无网络 `MatrixEffectSimpleDemo`：使用固定 seed 的内置柔和光团、响应式网格、30 FPS、1.5 DPR 和 3500 cells 上限，提供稳定的 4:3/16:9 容器与非装饰 Canvas 无障碍名称。
- 把完整 `MatrixEffectDemo` 和简单 Demo 接入详情页的两个静态映射；`generateStaticParams()` 自动包含新组件 ID。
- 将首页预览映射类型改为 `Partial<Record<ComponentId, React.ComponentType>>`，与调用端已有的“缺少预览则跳过”逻辑一致，避免新增组件 ID 被类型系统强制接入首版非目标的首页组件墙。
