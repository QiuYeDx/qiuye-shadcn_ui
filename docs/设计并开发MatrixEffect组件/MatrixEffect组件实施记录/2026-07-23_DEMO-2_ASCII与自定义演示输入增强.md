# 工作包 DEMO-2：ASCII 与自定义演示输入增强

## 基本信息

- 日期：2026-07-23
- 状态：已完成
- 对应执行计划工作包：`DEMO-2 ASCII 与自定义演示输入增强`

## 本次实现内容

- ASCII 字符集输入框初始值改为组件默认字符集 `" .:-=+*#%@"`，新增组件默认、精细层次、块元素和圆点四套 Demo 级预设，并保留直接编辑任意字符串的能力。
- ASCII 与自定义场景新增共享输入源控件，支持旋转星旋、流动光团、呼吸圆环、流动波浪、静态星旋和本地图片。
- 新增四个模块级稳定动态 Source；全部通过 `MatrixProceduralSource.time` 推进，不创建额外 rAF、定时器或 React 逐帧状态。
- 动态场提供 `playing` 开关，静态示例与上传图片自动禁用；核心仍负责 Reduced Motion、离屏暂停和帧率门控。
- 上传图片直接把 `File` 交给图片 Source adapter；Demo 不上传、不持久化、不创建 object URL，并显示当前文件名。
- ASCII 默认使用旋转星旋，自定义管线默认使用呼吸圆环，避免两个场景继续显示同一静态构图。
- 隐藏文件输入使用带重要级尺寸覆盖的 `sr-only` 样式，保留无障碍语义且不再扩大移动端文档宽度。
- ASCII 新增源色夜幕、荧光终端、琥珀终端、雾灰和蓝图五套配色预设，并提供字符色、背景色两个取色器；源图色模式下字符色控件禁用但保留固定色值。

## 接口或数据结构变化

- 新增站点 Demo 内部类型 `MatrixDemoPresetSourceId`、`MatrixDemoSourceId` 与预设元数据，不从 MatrixEffect registry item 导出。
- Demo 输入增强本身不新增公共枚举或 Source 类型；后续验收 fix 将 ASCII 默认采样格比例从 `0.6` 修正为 `1`，详见 `fix/2026-07-23_MatrixEffect_ASCII比例与动态星旋修复.md`。
- 未新增 npm 依赖，`package.json` 与 `pnpm-lock.yaml` 未修改。

## 与设计文档的一致性

- 是否偏离设计：否。
- 字符集和形状预设只在站点 Demo 内实现；组件仍通过通用 `characters`、`source` 和 `playing` 表达这些能力。
- 配色预设同样只属于 Demo，所有控件最终仍直接写入公共 `colorMode`、`color` 和 `backgroundColor` Props。
- 动态场复用核心调度与时间语义，上传图片复用现有 File/Blob Source 生命周期，没有引入第二条动画循环、远程代理或持久化。

```text

```

结果：

- pnpm `8.7.0` 下 Registry 正式同步与 dry-run 均为 14 个 item、0 个文件变化。
- 全量 ESLint、TypeScript、定向 Prettier 和生产构建通过；Next 成功生成 21 个静态页面。
- 真实浏览器确认 ASCII 初始字符集完整保留前导空格，输入源菜单含旋转/静态星旋；ASCII 与自定义场景均可使用旋转星旋。
- 四个程序化 Source 的间隔帧截图哈希均变化；旋转星旋暂停后连续帧哈希一致，恢复后再次变化。
- 5 套 ASCII 配色预设均可用；独立改色后显示“自定义”，源图色模式禁用字符色但保留其值，切回固定色后恢复可编辑。

- 本地 `source.webp` 选择后显示文件名、输入源显示“本地图片”、播放开关禁用。ASCII 配色交互未产生新的 warning/error；Next 开发工具仍显示既有 `CodeBlock` 明暗主题 SSR 水合属性不一致。
