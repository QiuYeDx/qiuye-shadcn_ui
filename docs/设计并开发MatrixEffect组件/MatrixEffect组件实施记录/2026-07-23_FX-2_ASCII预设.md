## 本次实现内容

- 新增 `createAsciiRenderer()`：把主信号按 `round(value * (glyphCount - 1))` 映射到从低到高密度排列的字符集，默认字符集为 `" .:-=+*#%@"`。
- 字符串按 Unicode code point 拆分，字符数组按 glyph token 快照；显式空字符串/空数组回退默认字符集并只在开发期告警一次，单字符集覆盖所有非透明格，包括 `value=0`。
- 支持固定色与 Source RGB；`a=0` 跳过，部分 Source Alpha 按 `a/255` 作为最终覆盖率，不引入额外低值或 Alpha 阈值。
- 字体只在 `prepare()` 中设置并调用一次 `measureText("M")` 缓存垂直基线；逐帧重新应用 Canvas 文本状态，逐格热路径不测量文本、不创建 cell 对象。
- Renderer 的字号、坐标和可选背景都使用 CSS px；默认提供 `cellAspectRatio=0.6` 与 `preferredFrameRate=30` 提示。
- 新增 `AsciiEffect`：要求显式 Source，固定执行 Luminance -> Invert -> Levels -> additionalTransforms，并只把 `backgroundColor` 映射为核心 `clearColor`。
- ASCII grid 按 auto/fixed mode 做字段级合并，默认 `cellSize=10`、`cellAspectRatio=0.6`、`maxCells=6000`；fixed 只指定 columns 时由核心按画布比例推导 rows。
- 使用 `forwardRef<MatrixEffectHandle>` 原样转发核心句柄，并通过字符内容签名稳定等价内联字符数组，避免无关父组件 render 重建 Renderer、重测字体或重置自适应 FPS。
- 从公共入口导出 `AsciiEffect`、`createAsciiRenderer`、`AsciiEffectProps` 和 `AsciiRendererOptions`，同步补齐设计契约、背景叠加顺序 JSDoc 与精确 Alpha/空白 glyph 表述。

## 接口或数据结构变化

- 新增 `AsciiRendererOptions` 与 `createAsciiRenderer()`。
- 新增 `AsciiEffectProps` 与 `AsciiEffect`；预设要求 `source`，且不暴露核心 `renderer`、`mapper`、`transforms`、`clearColor` 和未合并的核心 `grid`。
- `AsciiRendererOptions.backgroundColor` 是高级 Renderer 层背景：在核心 `clearColor` 后以 source-over 绘制；预设自身只提供一个 `backgroundColor` 清屏入口。
- ASCII 默认固定色为 `#71717a`，默认字重 `400`、`fontScale=1`，使用跨平台等宽字体栈。

## 与设计文档的一致性

- 是否偏离设计：否。
- 实现保持 Canvas 2D、TypedArray 整帧管线和 Source/Mapper/Transform/Renderer 分层，没有新增依赖、DOM 单元格、指针监听或第二条动画循环。
- 复审后修正了显式空字符集告警、低值/Alpha 阈值文档歧义、Renderer 背景层叠说明和等价内联字符数组 memo；这些修正使实现与既有设计一致，没有改变冻结的公共契约。
- Demo、站点接入和 Registry 继续分别保留在 DEMO-1、SITE-1、REG-1，不在 FX-2 提前混入。

结果：

- 当前 pnpm 为 8.7.0；未新增依赖，`package.json` 与 `pnpm-lock.yaml` 未修改。
- 全量 ESLint、TypeScript、Prettier、`git diff --check` 和生产构建通过；Next 成功生成 20 个正式页面。
- 31 项纯算法/Mock Canvas 断言通过：覆盖字符规范化与快照、空/单字符、索引边界、固定色/Source RGB、透明与部分 Alpha、字体缓存、CSS px/DPR、背景范围和无逐格测量。
- 临时类型契约断言通过：`source` 必填；`renderer`、`mapper`、`transforms`、`clearColor` 被拒绝；auto/fixed grid 分支与 ref 精确类型正确。
- 无 DOM 的 SSR 导入断言通过，`AsciiEffect` 和公共入口不会在服务端模块初始化时访问浏览器 API。
- 真实 Chromium 临时路由验证通过：`contain + invert` 的外侧四分之一区域 Alpha 总和为 0；单字符 `@` 在反相后 `value=0` 的非透明主体仍绘制。
- 固定色输出均值为约 `[0, 255, 102]`，Source 色输出均值为约 `[0, 96, 255]`；部分 Alpha 输出最大 Alpha 为 128；显式空字符串回退后产生非透明字符并在每次页面加载只告警一次。
- `{ mode: "fixed", columns: 100 }` 在 `1000 x 500` 容器中解析为 `100 x 30`；DPR 2 且 `maxDpr=1.5` 时，`240 x 120` CSS Canvas 的 backing store 为 `360 x 180`。
- 等价内联字符数组连续重渲染的 Source 重绘增量和字体测量增量均为 0；ref 指向 Canvas，两次同步 `invalidate()` 合并为一次重绘。
- Invert 后 additional Transform 观测值为 `0.498039186000824`；动态 ASCII 在 auto 模式下实测约 29.6 到 30.0 FPS；浏览器控制台无错误。
- 已人工检查完整页面截图：字体基线稳定，字符未被异常裁切，透明留白、固定色、Source 色、半透明、动态场与 100 列纵横比例均可辨认。
- 两轮独立只读复审覆盖公共 API/Omit/JSDoc、Canvas 状态、Alpha、背景、热路径、grid/transform/ref、Renderer memo 与导出；修复后无遗留 P0-P3 问题。
