# 工作包 FE-2：动态调度、暂停与错误韧性

## 基本信息

- 日期：2026-07-22

## 本次实现内容

- 把 FE-1 的一次性 dirty 绘制扩展为单一 rAF 调度器，以对象租约和 `frameInProgress` 保证 Strict Mode、同步重入和连续播放期间最多只有一条调度链。
- 明确 drawable、continuous 和 one-shot 三层调度条件；页面隐藏、启用中的离屏暂停、零尺寸和错误属于硬阻塞，`playing=false`、Reduced Motion freeze 和静态 Source 只停止连续播放。
- 记录 dirty generation 的最后尝试值，静态 loading/idle 不会在 rAF 尾部忙轮询；Source 状态变化或新的 `invalidate()` 才会唤醒下一次尝试。
- 实现固定 30/60 FPS deadline 门控、0.5ms 容差和大间隔不补帧；动态 `invalidate()` 不绕过连续帧上限，运行时切换帧率也从最近成功帧建立新 deadline。
- 实现 auto FPS 时间归一化 EMA：Renderer 30/60 提示、2 秒压力降档、10 秒稳定窗口、20 秒升级冷却，以及暂停清样本、身份变化重建控制器的规则。
- 实现事务式有效播放时钟；只有完整提交的连续帧推进 `time`，`time` 与 `deltaTime` 使用同一个最大 100ms 增量，Source epoch 切换归零，暂停恢复首帧和一次性重绘使用 `deltaTime=0`。
- 接入 document visibility、IntersectionObserver 和 `prefers-reduced-motion` 运行时监听；离屏 root margin 固定为 `128px 0px`，IO 缺失或构造/observe 失败时 fail-open，MediaQueryList 同时兼容现代和 legacy 监听 API。
- 使用环境租约成对清理 rAF、visibility、IO 和 MediaQueryList；无效策略下的 IO/MQL 观测仍更新缓存，但不会无意义重置播放时间或自适应样本。
- 在 staging 提交前后同时校验 pipeline、playback generation 和 Source adapter；旧 generation 的 Source/Mapper/Transform/Renderer 同步重入或异常只返回 stale，不会锁死新配置。
- 保留 FE-1 的结构化错误锁、最后成功画面和 fallback 语义；发生当前管线错误后取消连续调度并保证 `onError` 只上报一次。
