# 工作包 FE-1：MatrixEffect 静态渲染核心

## 基本信息

- 日期：2026-07-22
- 状态：已完成
- 对应执行计划工作包：`FE-1 MatrixEffect 静态渲染核心`

## 本次实现内容

- 创建 `MatrixEffect` 客户端组件，使用稳定根容器和可见 Canvas，并支持根节点属性、Canvas class、fallback 与 Canvas 无障碍语义。
- 实现响应式 `columns x rows` 与 fixed 网格解析，覆盖 cell aspect ratio、`maxCells`、极端长宽比和 16,384 单轴保护。
- 实现 DPR cap、4,194,304 backing-store 像素预算、16,384 单轴预算，以及 CSS 坐标到真实 backing store 的双轴缩放。
- 关闭 Source -> RGBA -> Mapper -> Transform -> finite/clamp -> Renderer 的静态完整管线，并复用 TypedArray 与上一成功帧信号。
- 使用内部 staging Canvas 事务绘制；只有完整成功帧才提交到可见 Canvas、推进 frame/history、进入 ready 并触发当前 Source epoch 的首次 `onReady`。
- 实现稳定 `MatrixEffectHandle`、实时 Canvas getter，以及合并重复请求和保留绘制期间重入请求的 dirty generation `invalidate()`。
- 按配置身份执行最小失效，使用 latest callback refs，并以 pipeline generation 和 adapter 身份阻止旧帧串入新配置或新 Source epoch。
- 实现结构化错误锁、错误单次上报、已有成功画面保留、初始 fallback，以及 recoverable/non-recoverable 错误恢复边界。
- 检测 Source、Mapper、Transform、Renderer 的非法 thenable 返回值，消费迟到 rejection，并轮换相应 Canvas、缓冲或上下文引用以隔离异步写入。
- 实现 Source 切换和 tainted Canvas 恢复；Source 运行时错误、Source epoch 切换和 Strict Mode 清理同时轮换采样 Canvas 与 procedural context。
- 实现 Renderer prepare 缓存和 Strict Mode dispose 微任务租约，保证资源只在替换或真正卸载后释放。
- 实现 ResizeObserver 和 window resize 降级，并能在 CSS 尺寸不变但设备 DPR 变化时重新分配与绘制。

## 接口或数据结构变化

- 公共入口新增 `MatrixEffect` 导出；已有 Source、Mapper、Transform、Renderer 与 Props 类型保持兼容。
- `MatrixSignalMapper`、`MatrixSignalTransform` 和 `MatrixRenderer` JSDoc 明确同步瞬时借用契约、staging Canvas 语义、配平 Canvas 状态和 Renderer 独占/幂等释放要求。
- 新增不从公共入口导出的确定性几何函数 `resolveMatrixGrid()`、`resolveMatrixCanvasMetrics()` 与 `normalizeMatrixSignalValues()`，供核心实现和验证复用。
- `playing`、`frameRate`、`pauseWhenOffscreen` 和 `reducedMotion` 仍由公共 Props 接收但不透传 DOM；它们将在 FE-2 接入连续调度。

## 与设计文档的一致性

- 是否偏离设计：否。
- 实现前补齐了极端网格、Canvas 像素/单轴预算、信号历史、staging 事务提交、同步插件契约、Source epoch、状态优先级和回调重入规则，再按这些规则实现。
- `prepare()` 只由 Canvas 尺寸、网格、DPR 或 Renderer 身份变化失效；Mapper、Transform、Source 和 clearColor 变化不会无意义重建 Renderer 缓存。
- FE-1 只使用一次性 rAF 消费静态 dirty 请求，时间字段保持 0；没有提前实现持续播放、自适应 FPS、离屏暂停或 Reduced Motion。
