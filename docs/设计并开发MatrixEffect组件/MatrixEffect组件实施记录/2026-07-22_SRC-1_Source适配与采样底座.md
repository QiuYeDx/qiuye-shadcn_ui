# 工作包 SRC-1：Source 适配与采样底座

## 基本信息

- 日期：2026-07-22
- 状态：已完成
- 对应执行计划工作包：`SRC-1 Source 适配与采样底座`

## 本次实现内容

- 创建仅供 MatrixEffect 内部管线使用的 Source adapter，统一图片、外部 Canvas 和程序化 Source 的 `animated`、`draw()` 与幂等 `dispose()` 协议。
- 实现 string URL、Blob/File、外部 `HTMLImageElement` 和 `ImageBitmap` 图片输入；图片异步状态通过内部回调唤醒后续重绘。
- 实现直接 Canvas、OffscreenCanvas 和 supplier 输入；supplier 每次绘制只求值一次，返回 `null` 或零尺寸时保持 idle。
- 实现程序化 Source 的同步调用协议、Canvas drawing state 隔离、current path 清理，以及意外 Promise 和同步异常的结构化错误。
- 实现 `cover`、`contain`、`fill`、position、smoothing、透明/固定背景和无效背景色的确定性行为。
- 实现内部 Image、Blob object URL 和监听器所有权；外部 Image、ImageBitmap、Canvas 与程序化资源始终由调用方持有。
- 使用 disposed 令牌和成对监听器清理阻止过期图片回调覆盖新 Source，保证 React Strict Mode 重复初始化/清理安全。
- 实现跨 realm DOM 资源识别，以及 `getImageData()` 安全异常到 `SOURCE_SECURITY_ERROR` 的异常安全分类。
- 在实现前补齐内部 adapter、采样错误、精确 fit、同步程序化 Source 与 tainted Canvas 恢复契约。

## 接口或数据结构变化

- 新增内部 `MatrixSourceAdapter`、`MatrixSourceDrawResult` 和 `MatrixSourceAdapterOptions` 契约；这些符号只供同目录核心模块使用，不从公共 `index.ts` 导出。
- 新增内部 `createMatrixSourceAdapter()`，创建时快照描述符配置，绘制时读取外部图片或 Canvas 的实时 backing 尺寸和像素。
- 新增内部 `resolveMatrixSourceDrawRect()` 纯几何函数，使用九参数 `drawImage()` 所需的源/目标矩形表达三种 fit。
- 新增内部 `isMatrixSourceSecurityError()` 与 `createMatrixSourceSampleError()`，供 FE-1 捕获 `getImageData()` 异常时生成公共 `MatrixEffectError`。
- 公共类型形状未变化；只加强 `MatrixProceduralSource.draw` 的 JSDoc，明确回调必须同步且不能异步持有采样上下文。

## 与设计文档的一致性

- 是否偏离设计：否。
- 原设计已经确定公共 Source 描述符，但没有定义异步图片如何通知核心、统一 draw 状态和 CORS 读取异常的内部桥接协议；本次先把这些规则补入设计再实现。
- `cover` 使用 position 控制源裁切焦点，`contain` 使用 position 控制目标留白对齐，`fill` 忽略 position 并拉伸铺满。
- 所有 Source 错误的 `recoverable=true` 表示更换 Source 或配置后可以重试，不允许核心对同一错误配置逐帧忙重试。
- 采样 Canvas 一旦 tainted 无法通过 `clearRect()` 恢复；该恢复动作归属 FE-1，并已写入设计与执行计划交接项。
