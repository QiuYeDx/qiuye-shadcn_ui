# 快速上手页面重构执行计划

## 使用方式

后续 Agent 开始工作时，先读取本执行计划和同目录下的《快速上手页面重构开发设计文档》。每次只认领一个小工作包，完成后更新进度台账并写入实施记录。

## 状态规则

- `未开始`：尚未编辑或验证。
- `进行中`：已有部分实现，但尚未完成验证或记录。
- `已完成`：代码和文档已同步，相关校验已运行或明确记录限制。
- `阻塞`：无法继续，需要外部输入。
- `废弃`：设计或用户要求变化后不再执行。

## 工作包说明

### PRE-1 现状梳理与设计

- 确认旧 `/cli` 页面结构、导航入口、registry 数据来源和包管理约束。
- 创建最终设计文档和执行计划。

### FE-1 重写快速上手页面

- 删除旧 tab 文档页面结构。
- 新建面向使用者的快速开始页面。
- 保留命令复制、包管理器切换、registry 配置和组件入口。
- 更新导航文案为“快速开始”。

### QA-1 校验与收尾记录

- 运行可行的 lint/build 校验。
- 记录验证结果和未完成事项。
- 更新进度台账。

## 执行顺序

1. 先完成 PRE-1，让页面目标和边界稳定。
2. 再做 FE-1，确保 `/cli` 兼容旧路径但视觉与文案彻底更新。
3. 最后做 QA-1，只把实际验证通过的内容标为已完成。

## 不可违反约束

- 不修改 `pnpm-lock.yaml`。
- 不用新版 pnpm 重写依赖锁文件；需要 pnpm 时优先使用 pnpm 8 系列。
- 不启动并遗留前端服务进程。
- 不破坏 `/cli` 路由。
- 页面不继续以“CLI 工具文档”为主要心智。
