# 官网首页与布局优化开发设计文档

创建日期：2026-06-05
更新日期：2026-06-05

## 需求摘要

当前 QiuYe UI 组件库官网的全局 layout 和首页视觉偏传统产品介绍页：常驻左侧应用侧边栏、渐变 Hero、统计数字、Feature 卡片和 CTA 区块占据主要篇幅。用户希望参考 shadcn/ui 官网的风格与样式，对组件库官网重新设计，尤其是在首页首屏展示已有自定义组件，让访问者能更快感知组件库本身的能力与审美。

参考对象：shadcn/ui 官网首页（https://ui.shadcn.com/，2026-06-05 访问）。它的核心结构是：

- 顶部横向导航，而不是应用型左侧侧边栏。
- 居中大留白 Hero，标题直接传达设计系统定位。
- 首屏下半部分使用组件预览墙展示按钮、表单、卡片、图表、通知等真实界面片段。
- 色彩以黑白灰和边框层次为主，少装饰、少渐变、重组件本体。

## 目标

- 将官网从“应用后台壳”调整为“组件库/设计系统官网壳”，首页优先展示组件价值。
- 首页首屏形成类似 shadcn/ui 的结构：顶部导航、居中 Hero、下方组件预览墙。
- 首页组件展示由现有 `lib/registry.ts` 的组件注册表驱动，避免重复维护组件清单。
- 使用 QiuYe UI 真实组件或轻量预览变体展示所有当前自定义组件。
- 保持 Next.js App Router、React 19、Tailwind CSS、shadcn/ui 现有技术栈。
- 保持现有组件详情页、CLI 页面和 registry 安装能力不破坏。
- 提供移动端、暗色模式和性能可控的设计方案。

## 非目标

- 不逐像素复刻 shadcn/ui 官网。
- 不改组件本身的 public API。
- 不重写 `public/registry/*.json` 的生成规则。
- 不新增后端服务、登录、收藏、远程搜索或埋点系统。
- 不在本阶段实现完整文档站搜索引擎；如实现搜索，仅限本地组件索引搜索。
- 不为了首页展示而强行加载所有重型 demo 的完整交互版本。

## 当前工程背景

项目当前是 Next.js App Router 项目，主要文件如下：

- `app/layout.tsx`：全局挂载 `ThemeProvider`、`SidebarProvider`、`AppSidebar`、`Header`、`Toaster`、Vercel Analytics 和 Speed Insights。
- `components/header.tsx`：顶部栏依赖 `useSidebar()`，主要展示首页、组件列表、CLI 工具、主题切换。
- `components/app-sidebar.tsx`：全局左侧侧边栏，包含首页、组件列表、CLI 工具。
- `app/page.tsx`：当前首页，包含渐变 Hero、统计数字、Feature 卡片和 CTA。
- `app/components/page.tsx`：组件列表页，使用 `getAllComponents()`、`getCategories()`、`searchComponents()` 展示组件卡片和复制 CLI 命令。
- `app/components/[id]/page.tsx`：组件详情页，已维护完整 demo 映射与简单 demo 映射。
- `app/components/[id]/simple-demos.tsx`：组件详情页的精简预览集合，可为首页预览提供参考，但不建议直接整包导入到首页。
- `lib/registry.ts`：组件元信息注册表，是首页组件墙的数据源。
- `lib/component-constants.ts`：组件 ID 枚举，目前包含 10 个自定义组件。
- `app/globals.css`：已有 shadcn/ui 风格主题 token、暗色模式 token 和全局基础样式。

当前已注册自定义组件：

- `responsive-tabs`
- `scrollable-dialog`
- `dot-glass`
- `image-viewer`
- `dual-state-toggle`
- `code-block`
- `typewriter`
- `markdown-renderer`
- `color-picker`
- `tour`

## 设计原则

- 组件优先：首屏视觉中心从营销文案转为组件预览。
- 黑白灰为主：沿用 shadcn/ui 的中性色、边框、留白和低阴影层次，不使用大面积彩色渐变背景。
- 信息密度适中：首页保持展示性，组件页保持检索效率。
- 真实但轻量：首页预览应尽量体现真实组件能力，但避免加载完整文档 demo 带来的首屏性能压力。
- 数据单源：组件名称、描述、分类、版本、tags、CLI 名称均来自 `componentRegistry`。
- 渐进增强：搜索、动画和主题切换可增强体验，但基础导航、组件链接和复制命令必须可靠。
- 响应式稳定：组件墙卡片使用明确尺寸、栅格跨度和溢出处理，避免 hover、异步内容或长文本造成布局跳动。

## 全局 Layout 设计

### 问题

当前 `app/layout.tsx` 对所有页面都包裹 `SidebarProvider`、`AppSidebar` 和依赖侧边栏状态的 `Header`。这让首页天然呈现应用后台的框架感，不适合组件库官网首页。

### 目标结构

新增站点级 shell：

```text
components/site/
  site-shell.tsx
  site-header.tsx
  site-mobile-nav.tsx
  site-search.tsx
```

`app/layout.tsx` 调整为：

```tsx
<ThemeProvider ...>
  <SiteShell>{children}</SiteShell>
  <Toaster />
  <Analytics />
  <SpeedInsights />
</ThemeProvider>
```

`SiteShell` 是客户端组件，用 `usePathname()` 判断是否需要 docs 型侧栏。第一阶段可以统一使用顶部导航，不再全局渲染 `AppSidebar`。

### AppSidebar 处置

`components/app-sidebar.tsx` 暂不删除，避免影响未来文档型布局。第一阶段从根 layout 中移除常驻引用即可。

若后续组件详情页内容增多，可引入 `DocsShell`：

```text
components/site/docs-sidebar.tsx
components/site/docs-shell.tsx
```

并只在 `/components/[id]` 或未来 `/docs/*` 中使用。

## Header 交互设计

### 桌面端

- 高度：`h-14` 或 `h-16`，与 shadcn/ui 官网接近。
- 背景：`bg-background/95 backdrop-blur supports-[backdrop-filter]:bg-background/60`。
- 底部边框：`border-b`。
- 内容最大宽度：`max-w-screen-2xl`，左右 `px-4 sm:px-6 lg:px-8`。
- 左侧品牌：`QiuYe UI`，可配一个简洁方形图标。
- 中间导航：Home、Components、CLI。
- 右侧：
  - `SiteSearch`：桌面端显示 `Search components...`。
  - GitHub 图标链接。
  - `ThemeToggle`。
  - `Install` 主按钮，跳转 `/cli`。

### 搜索行为

第一阶段推荐实现轻量本地搜索：

- 数据源：`getAllComponents()` 序列化出的组件名称、描述、tags、cliName。
- 打开方式：点击 header 搜索框，或快捷键 `⌘K` / `Ctrl+K`。
- 展示方式：用现有 `Dialog` 或 `Popover` 呈现结果列表。
- 结果项点击跳转 `/components/[cliName]`。
- 空态文案：`没有找到组件`。

如果搜索不在第一阶段实现，Header 中不要放一个不可用的假搜索框；可以先只在 `/components` 页面保留搜索。

### 移动端

- 隐藏桌面导航和搜索框。
- 展示品牌、搜索图标、主题切换、菜单图标。
- 菜单使用 `Sheet`，每个导航项点击后关闭。
- 搜索图标点击可直接跳转 `/components` 并聚焦组件页搜索框；若暂不实现跨页聚焦，则只跳转 `/components`。

## 首页设计

### 页面结构

```text
HomePage
  SiteHeader
  HeroSection
  HomeComponentWall
  ComponentDirectoryBand
  InstallBand
  SiteFooter
```

第一屏重点是 `HeroSection + HomeComponentWall`，后续区块保持克制。

### HeroSection

设计方向：

- 背景使用纯 `bg-background`，不使用大面积渐变。
- 内容居中，最大宽度约 `max-w-4xl`。
- 顶部 pill：`QiuYe UI Registry` 或 `Built on shadcn/ui`。
- H1 建议使用品牌名或明确类别：

```text
QiuYe UI
```

或：

```text
The Component System for Your Next App
```

中文版本建议：

```text
QiuYe UI
```

副标题：

```text
一组基于 shadcn/ui 构建的高质量自定义组件。复制、安装、组合，然后把它变成你自己的设计系统。
```

CTA：

- 主按钮：`Browse Components` / `浏览组件`，跳转 `/components`。
- 次按钮：`CLI Install` / `CLI 安装`，跳转 `/cli`。

Hero 不再展示统计数字，不再展示“为什么选择”类 Feature 卡片。组件数量可放在 pill 或副标题中，例如“10 components ready to install”。

### HomeComponentWall

首页组件墙是核心变化。它应展示当前所有 QiuYe UI 自定义组件，并链接到对应详情页。

桌面布局：

- 使用 CSS Grid，而不是瀑布流 JS。
- 建议 `grid-cols-4`，每个卡片使用 `row-span-*` 或固定高度形成错落节奏。
- 最大宽度 `max-w-screen-2xl`。
- 上移到首屏下半部，让用户不滚动也能看到组件墙顶部。
- 底部可使用轻量遮罩淡出，暗示还有更多内容，但不要遮挡关键信息。

移动端布局：

- 单列或双列，优先单列。
- Hero 文案减少上边距，确保 390x844 视口能看到至少 1 个组件预览卡片。
- 卡片内长描述最多两行，按钮和标签不挤压。

组件墙数据模型建议：

```tsx
export type HomePreviewSize = "wide" | "tall" | "square" | "compact";

export interface HomeComponentPreviewConfig {
  id: ComponentId;
  size: HomePreviewSize;
  featured?: boolean;
  preview: React.ComponentType;
}
```

元信息从 registry 补齐：

```tsx
interface HomeComponentPreviewItem extends HomeComponentPreviewConfig {
  name: string;
  description: string;
  category: string;
  cliName: string;
  tags: string[];
}
```

### 组件预览映射

建议新增：

```text
components/home/
  home-hero.tsx
  home-component-wall.tsx
  home-preview-card.tsx
  home-component-previews.tsx
lib/home-component-preview-config.ts
```

预览策略：

| 组件 | 首页预览方式 | 备注 |
|---|---|---|
| Responsive Tabs | 使用真实 `ResponsiveTabs` 做 3 个短 tab | 低成本，可交互 |
| Scrollable Dialog | 展示一个 compact dialog mock + 打开按钮 | 可使用真实 Dialog，但默认不打开 |
| Dot Glass | 使用真实 `DotGlass` 覆盖一个黑白内容面板 | 强视觉组件，建议 featured |
| Image Viewer | 展示真实图片预览，点击可进入详情页 | 首页不建议打开完整灯箱 |
| Dual State Toggle | 使用真实 `DualStateToggle` 展示 2-3 个图标按钮 | 低成本，可交互 |
| Code Block | 使用真实 `CodeBlockPanel` 或轻量静态代码面板 | 注意不要撑高 |
| Typewriter | 使用真实 `Typewriter` 展示短文案 | 低成本 |
| Markdown Renderer | 用轻量 Markdown 结果片段或静态 facsimile | 避免首屏加载 Mermaid 等重型能力 |
| Color Picker | 使用真实 `ColorPicker` 或 swatch 组合 | 可交互但要控制弹层 |
| Tour | 使用静态产品界面 + spotlight mock | 不在首页启动真实 Tour |

首页预览卡片统一包含：

- 顶部区域：组件名称、分类 badge。
- 中间区域：预览。
- 底部区域：一句短描述或 tags。
- 整卡点击进入 `/components/[cliName]`。
- 卡片内部若有交互控件，需要阻止点击冒泡，避免误跳转。

### 视觉细节

- 卡片使用现有 `Card` 风格：`rounded-lg border bg-card shadow-sm`。
- 阴影控制在 `shadow-sm` 或 hover `shadow-md`，不使用强烈投影。
- 页面背景用 `bg-background`，组件墙区域可以用 `bg-muted/30` 或 `border-y` 做区分。
- 字号层级：
  - Hero H1：`text-5xl sm:text-6xl lg:text-7xl`，移动端至少 `text-4xl`。
  - Hero 描述：`text-base sm:text-xl`。
  - 卡片标题：`text-sm` 或 `text-base font-medium`，不要用 hero 级大标题。
- 图标统一使用 `lucide-react`。
- 字间距保持默认，不使用负 letter-spacing。
- 不使用装饰性 orb、bokeh、复杂 SVG 背景。

### 动画

现有首页大量使用 `motion` 入场、hover scale 和统计滚动。新首页动画应减少：

- Hero 入场可保留轻微 `opacity/y`。
- 组件墙卡片可以 stagger 入场，但时长要短。
- hover 使用 `translate-y-1` 或边框变化，不使用明显 scale。
- 尊重 `prefers-reduced-motion`，必要时禁用复杂动画。

## 组件列表页优化方向

`app/components/page.tsx` 当前功能完整，但视觉可与新首页统一：

- 页面头部改为更紧凑的 docs directory 风格。
- 搜索和包管理器选择可放在同一行，移动端折行。
- 分类 tabs 保留，但视觉减少高度和动效。
- 组件卡片减少营销式描述，强化：
  - 组件名
  - 分类
  - 简短描述
  - CLI 命令
  - 查看详情
- 卡片 hover 保持克制，不大幅位移。

此部分可作为第二阶段，不阻塞首页首屏优化。

## 组件详情页影响

`app/components/[id]/page.tsx` 已经具备：

- 组件详情 header。
- 快速预览。
- 安装命令。
- 导入代码。
- Demo / API / Dependencies tabs。

本需求不要求大改详情页。需要注意：

- 如果移除全局 `AppSidebar`，详情页顶部返回按钮仍可正常工作。
- 详情页的 sticky 侧栏 `top-24` 需要根据新 Header 高度校准。
- `DemoPreview` 和首页预览不要共用完整 demo 映射，避免首页 bundle 变重。

## 数据与状态设计

### Registry 数据

首页组件墙只读取 `getAllComponents()`，不直接读 `public/registry/registry.json`。

排序建议：

1. Featured 组件优先：`dot-glass`、`code-block`、`markdown-renderer`、`tour`。
2. 交互基础组件穿插：`responsive-tabs`、`dual-state-toggle`、`color-picker`。
3. 展示/容器组件补齐：`scrollable-dialog`、`image-viewer`、`typewriter`。

排序配置放在 `lib/home-component-preview-config.ts`，没有配置的组件自动追加到末尾，保证新增组件不会从首页消失。

### 包管理器状态

当前组件页和详情页分别存在 package manager 状态逻辑。首页如果需要展示安装命令，不建议再复制一套状态。

第一阶段首页 CTA 只跳转 `/cli`，不展示可复制命令。

若要展示命令，建议后续抽取：

```text
hooks/use-package-manager.ts
components/site/package-manager-selector.tsx
components/site/copy-install-command.tsx
```

### 搜索状态

如果实现 `SiteSearch`：

- 搜索 query 存在组件本地状态。
- 结果由序列化组件索引同步过滤。
- 不写 URL query。
- 不持久化。

## 可访问性

- Header 当前路由使用 `aria-current="page"`。
- 移动端菜单按钮有明确 `aria-label`。
- 搜索 Dialog 打开后焦点进入输入框，关闭后回到触发器。
- 组件墙卡片作为链接时，卡片内交互控件需要可键盘访问，并阻止链接嵌套违规。
- 图片预览必须有 `alt`。
- 动画尊重 `prefers-reduced-motion`。
- 颜色对比使用现有 shadcn token，暗色模式逐项检查。

## 性能设计

- `app/page.tsx` 尽量保持服务端组件，读取 registry 后把必要元信息传给客户端预览墙。
- 首页预览组件只导入需要的轻量组件。
- 不从 `app/components/[id]/simple-demos.tsx` 直接导入全部 demo，因为它会把大量组件和依赖一起拉进首页 bundle。
- 重型预览采用静态 facsimile：
  - `MarkdownRenderer` 不在首页启用 Mermaid。
  - `Tour` 不在首页运行定位/遮罩逻辑。
  - `ImageViewer` 不在首页默认打开灯箱。
- 图片优先使用本地或稳定远程资源，并设置固定宽高或 aspect ratio。
- 组件墙卡片设置明确高度，避免 hydration 后布局跳动。

## 实施阶段建议

### Phase 1：Layout 与首页首屏

- 新增 `SiteShell` 和 `SiteHeader`。
- 从根 layout 移除全局 `AppSidebar`。
- 重写 `app/page.tsx` 为 shadcn/ui 风格首页结构。
- 新增首页 Hero 和组件墙。
- 所有已注册组件在首页可见并能进入详情页。

验收标准：

- 1440x1100 首屏能看到 Header、Hero 和组件墙顶部。
- 390x844 首屏能看到 Header、Hero CTA 和至少一个组件预览。
- 首页不再出现全局左侧侧边栏。
- `/components`、`/components/[id]`、`/cli` 可正常访问。

### Phase 2：站内搜索与组件目录精修

- 新增 `SiteSearch`。
- 组件页视觉同步为更克制的 docs directory 风格。
- 抽取 package manager 状态 hook，消除重复逻辑。

验收标准：

- Header 搜索能搜索组件并跳转详情页。

- 桌面与移动端截图无明显重叠、溢出、空白断层。

## 风险与处理

### 首页 bundle 变大

风险：为了展示所有组件而导入完整 demo，导致首页加载变慢。

处理：建立专用轻量 preview，不直接导入详情页完整 demo。

### 移除全局 Sidebar 后页面宽度变化

风险：组件页和 CLI 页原本在 `SidebarInset` 下展示，移除后宽度和间距可能变化。

处理：为各页面统一使用 `container mx-auto px-4 sm:px-6 lg:px-8`，并在视觉 QA 中检查。

### Header 搜索过早实现导致复杂度上升

风险：搜索 Dialog、快捷键、焦点管理会扩大第一阶段范围。

处理：第一阶段可不实现 Header 搜索，或只做点击跳转 `/components`；搜索作为 Phase 2。

### 首页交互卡片与链接冲突

风险：整卡链接和内部按钮、tabs、color picker 交互冲突。

处理：预览卡片不要使用嵌套 `<a>`；使用卡片底部显式详情按钮，或在交互区域阻止冒泡。

### 视觉过度接近 shadcn/ui

风险：过度复刻导致缺少 QiuYe UI 自身识别度。

处理：结构学习 shadcn/ui，内容和预览全部来自 QiuYe UI 自有组件，文案保持中文组件库定位。
