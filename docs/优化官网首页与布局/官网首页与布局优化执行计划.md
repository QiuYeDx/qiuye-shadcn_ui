## 使用方式

每次继续开发前先阅读：

- `docs/优化官网首页与布局/官网首页与布局优化开发设计文档.md`
- 本执行计划

然后从进度台账中选择一个 `未开始` 或 `进行中` 的工作包，保持实现范围和设计文档一致。若实现中发现设计假设不成立，需要同步更新设计文档或新增 feat/fix 记录。

## 状态规则

- `未开始`：尚未编码。
- `进行中`：已经开始实现但未完整验证。
- `已完成`：代码完成并通过记录的验证。
- `阻塞`：需要用户输入或外部条件。
- `废弃`：不再执行。

## 依赖顺序

1. 先完成 `LAYOUT-1`，让全站从应用侧边栏壳切到官网顶部导航壳。
2. 再完成 `HOME-1` 和 `HOME-2`，关闭首页首屏的最小可见闭环。
3. `FE-SEARCH-1` 可与 `LAYOUT-1` 同步完成，但若影响范围变大，应后置。
4. `QA-1` 必须在 Phase 1 代码完成后执行。
5. `FE-COMP-1` 与 `DOC-1` 是后续增强，不阻塞 Phase 1。

## 设计约束

- 首页组件数据必须来自 `lib/registry.ts`。
- 首页预览不能直接导入详情页完整 demo 集合。
- 不修改组件 public API。
- 不修改 CLI 命令格式。
- 不删除 `components/app-sidebar.tsx`，仅从全局根布局移除。
- 移动端首屏必须能看到 Hero CTA 和至少一个组件预览。
- 不使用大面积渐变、装饰 orb、复杂 SVG 背景。
